# User, Builder Profile, Scout Profile 테이블 후보

> 이 문서는 DB 스키마 초안 문서다. 실제 SQL, Supabase migration, RLS 정책, API, 프론트엔드 구현은 작성하지 않는다. 테이블명과 필드명은 후보이며 최종 확정이 아니다.

## 1. User와 인증 사용자 정보의 관계

BuildMap은 로그인 또는 최소 계정 기반 사용을 전제로 한다. Supabase Auth를 사용할 가능성은 고려하지만, 이번 단계에서 실제 연동 구조를 확정하지 않는다.

### 후보 구조

| 후보 | 설명 | 장점 | 단점 | 권장 방향 |
|---|---|---|---|---|
| Auth 사용자만 참조 | Supabase Auth의 사용자 식별자를 직접 참조한다. | 단순하다. | 제품용 사용자 메타데이터 확장이 제한될 수 있다. | 추가 검토 필요 |
| 별도 users 후보 | 인증 사용자 참조와 제품 사용자 정보를 분리한다. | 확장성이 좋다. | 1차 구조가 커질 수 있다. | Builder/피드백 요구를 보고 결정 |

## 2. Builder Profile 후보

### 제품상 원칙

Builder는 프로젝트를 만든 사람이 아니라 프로젝트를 성장시키는 사람이다. Builder Profile은 이 사람이 어떤 프로젝트를 어떤 관점으로 운영하는지 외부와 내부 화면에서 식별하기 위한 제품 정보다.

### 1차 포함 여부

1차 필수다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| 사용자 참조 | 인증 사용자와 Builder Profile을 연결한다. | 필수 | 시스템 | 시스템 | 비공개 | 가입 |
| 공개 이름 | 공개 페이지와 카드에서 보이는 이름이다. | 필수 후보 | Builder | Builder | 공개 가능 | 공개 페이지 |
| 역할 태그 | 창업자, 학생 메이커, 개발자, 디자이너, PM, 비개발자, 바이브코딩 유저 등 | 선택/강력 권장 | Builder | Builder | 공개 가능 | 가입, 프로필 |
| 관심 분야 | 프로젝트 탐색과 자기소개 보조 정보다. | 선택 | Builder | Builder | 공개 가능 | 프로필 |
| 소개 | Builder가 자신을 설명하는 짧은 문장이다. | 선택 | Builder | Builder | 공개 가능 | 공개 페이지 |
| 생성 시점 | 프로필 생성 시점이다. | 필수 후보 | 시스템 | 시스템 | 비공개 | 내부 관리 |

### 공개 가능 정보

- 공개 이름
- 역할 태그
- 관심 분야
- 소개
- 공개 프로젝트 목록

### 비공개 정보

- 인증 식별자
- 이메일
- 내부 설정
- 비공개 프로젝트 목록

### 관계

- Builder Profile은 여러 Project를 소유할 수 있다.
- Change Card 작성/승인 주체가 될 수 있다.
- Feedback Request 생성 주체가 될 수 있다.

## 3. Scout Profile 후보

### 제품상 원칙

Scout는 초기에는 채용 담당자가 아니다. Scout는 프로젝트 발견자, 피드백 제공자, 테스터, 협업자, 멘토, 팀원 탐색자다.

### 1차 포함 여부

1차 선택 데이터다. 초기에는 별도 Scout Profile 없이 로그인 사용자가 공개 프로젝트를 보고 피드백을 남기는 구조로 시작할 수 있다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| 사용자 참조 | 로그인 사용자와 Scout Profile을 연결한다. | 선택 | 시스템 | 시스템 | 비공개 | Scout 설정 |
| 관심 분야 | 어떤 프로젝트를 보고 싶은지 나타낸다. | 선택 | 사용자 | 사용자 | 비공개 또는 공개 선택 | 탐색 |
| 탐색 목적 | 피드백, 테스터, 협업, 멘토링 등 목적이다. | 선택 | 사용자 | 사용자 | 선택 공개 | Scout 탐색 |
| 저장한 프로젝트 연결 | Project Save/Follow와 연결될 수 있다. | 선택 | 사용자 | 사용자 | 비공개 기본 | 저장 목록 |

### 제외할 정보

- AI 역량 점수
- Project DNA
- 헤드헌팅 계약 정보
- 지원자 관리 정보
- 투자자 인증
- 채용 후보자 등급

### 추가 검토 필요

Scout Profile을 만들지 않아도 로그인 기반 피드백과 프로젝트 저장이 가능하다면 1차에서 제외할 수 있다.
