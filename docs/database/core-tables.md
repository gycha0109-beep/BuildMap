# 핵심 테이블 후보

> 이 문서는 DB 스키마 초안 문서다. 실제 SQL, Supabase migration, RLS 정책, API, 프론트엔드 구현은 작성하지 않는다. 테이블명과 필드명은 후보이며 최종 확정이 아니다.

## 1. 핵심 테이블 후보 목록

6단계에서 가장 핵심이 되는 테이블 후보는 다음이다.

- `users` 또는 auth users 참조
- `projects`
- `change_cards`

이 중 BuildMap의 제품 정체성을 가장 강하게 담는 후보는 `change_cards`다.

---

## 2. users 또는 auth users 참조

### 제품상 역할

사용자를 식별하고 Builder Profile, 피드백 작성자, 선택적으로 Scout Profile과 연결하는 기준이다.

### 원천 데이터 여부

원천 데이터다. 단, Supabase Auth를 사용할 경우 인증 원천은 Supabase Auth가 되고, 제품 프로필 테이블은 이를 참조하는 구조가 될 수 있다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| 인증 사용자 참조 | 로그인 계정과 제품 데이터를 연결한다. | 필수 | 시스템 | 시스템 | 비공개 | 가입, 로그인 |
| 공개 이름 후보 | 공개 페이지나 피드백에 표시될 수 있다. | 선택 | 사용자 | 사용자 | 공개 가능 | 공개 페이지, 피드백 |
| 이메일 후보 | 인증 또는 알림에 사용할 수 있다. | 선택/인증 의존 | 시스템 | 사용자/시스템 | 비공개 | 계정 |
| 생성 시점 | 가입 시점을 나타낸다. | 필수 후보 | 시스템 | 시스템 | 비공개 | 관리자성 검토 |

### 추가 검토 필요

- 별도 `users` 테이블을 둘지, Supabase Auth 사용자만 참조할지 결정이 필요하다.
- 공개 이름은 Builder Profile에 둘 수도 있다.

---

## 3. projects 테이블 후보

### 제품상 역할

Project는 제품 소개글이 아니다. Project는 문제 정의, 가설, 변화 카드, 피드백, 공개 뷰를 묶는 판단 흐름 컨테이너다.

### 원천 데이터 여부

원천 데이터다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| 소유 Builder 참조 | 프로젝트의 주 책임 Builder를 연결한다. | 필수 | 시스템/Builder | 소유자 변경은 후순위 | 일부 공개 가능 | 프로젝트 생성, Builder 대시보드 |
| 프로젝트명 | 프로젝트를 식별하는 이름이다. | 필수 | Builder | Builder | 공개 가능 | 프로젝트 생성, 공개 페이지 |
| 한 줄 정의 | 프로젝트를 짧게 설명한다. | 필수 | Builder | Builder | 공개 가능 | 프로젝트 생성, 공개 페이지 |
| 현재 진행 상태 | 아이디어, 제작 중, 테스트 중 등 진행 단계를 나타낸다. | 필수 | Builder | Builder | 공개 가능 | 대시보드, 공개 페이지 |
| 공개 상태 | 비공개, 링크 공개, 전체 공개를 나타낸다. | 필수 | Builder | Builder | 권한 판단에 사용 | 공개 전환 |
| 현재 필요한 것 요약 | 외부에 보여줄 현재 필요 도움의 요약이다. | 선택에 가깝지만 강력 권장 | Builder | Builder | 공개 가능 | 공개 페이지, 카드 그리드 |
| 최근 업데이트 시점 | 탐색 정렬과 활동 표시의 기본값이다. | 필수 후보 | 시스템 | 시스템 | 공개 가능 | 카드 그리드 |
| 생성 시점 | 프로젝트 생성 기록이다. | 필수 | 시스템 | 시스템 | 공개 가능 | 대시보드 |

### 상태 필드 후보

Project 진행 상태와 공개 상태는 반드시 분리한다.

진행 상태 후보:

- 아이디어
- 제작 중
- 테스트 중
- 베타
- 운영 중
- 일시 중단
- 종료됨

공개 상태 후보:

- 비공개
- 링크 공개
- 전체 공개

### 주요 관계

- Builder Profile 하나가 여러 Project를 가질 수 있다.
- Project 하나는 여러 Problem Definition, Hypothesis, Rough Note, Change Card, Feedback Request와 연결될 수 있다.
- 공개 프로젝트 페이지는 Project의 공개 뷰로 파생된다.

### 삭제 또는 보관 정책 후보

1차에서는 물리 삭제보다 보관 또는 비공개 전환을 우선 검토한다. 프로젝트가 삭제되면 Change Card와 Feedback의 문맥이 사라질 수 있다.

### 추가 검토 필요

- 여러 Builder가 참여하는 팀 프로젝트는 2차 확장으로 둘 수 있다.
- Project 상태 변경 이력을 별도 저장할지, Change Card로 충분히 추적할지 실제 구현 전 검토한다.

---

## 4. change_cards 테이블 후보

### 제품상 역할

Change Card는 프로젝트에서 발생한 하나의 의사결정 단위다. BuildMap의 핵심 원천 기록이며 Decision Timeline의 원천이다.

### 원천 데이터 여부

핵심 원천 데이터다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| project 참조 | 어떤 프로젝트의 판단인지 연결한다. | 필수 | 시스템/Builder | 변경 제한 | 비공개/공개 판단에 영향 | Change Card, Timeline |
| 작성 Builder 참조 | 카드를 작성 또는 승인한 Builder를 연결한다. | 필수 | 시스템 | 제한적 수정 | 일부 공개 가능 | Timeline, 내부 기록 |
| 유형 | 문제 발견, 가설 반박, 기능 제거 등 카드 성격이다. | 필수 | AI 제안/Builder 선택 | Builder | 공개 가능 | 카드 검토 |
| 제목 | Timeline에서 식별되는 짧은 제목이다. | 필수 | AI 제안/Builder | Builder | 공개 가능 | Timeline |
| 구조화 요약 | 판단 흐름을 한눈에 이해하기 위한 요약이다. | 필수 | AI 제안/Builder | Builder | 공개 가능 | Timeline, 공개 페이지 |
| 근거 | 판단의 기반이 되는 피드백, 실험, 관찰이다. | 강력 권장 | AI 제안/Builder | Builder | 선택 공개 | 카드 검토 |
| 판단 | Builder가 내린 결정이다. | 강력 권장 | AI 제안/Builder | Builder | 선택 공개 | 카드 검토 |
| 변경 내용 | 무엇이 바뀌었는지 설명한다. | 강력 권장 | AI 제안/Builder | Builder | 선택 공개 | Timeline |
| 다음 확인 사항 | 다음 검증 또는 행동을 남긴다. | 강력 권장 | AI 제안/Builder | Builder | 선택 공개 | 대시보드 |
| 작업 상태 | 초안, 수정 중, 승인됨, 보류됨을 구분한다. | 필수 | 시스템/Builder | Builder | 공식 반영 판단 | 카드 검토 |
| 공개 상태 | 내부 전용, 공개 가능, 공개됨, 민감 정보 포함을 구분한다. | 필수 | Builder | Builder | 공개 페이지 판단 | 공개 전환 |
| 중요도 | 일반 또는 핵심 전환 후보를 구분한다. | 선택/강력 권장 | AI 제안/Builder | Builder | 공개 가능 | Timeline |
| 승인 시점 | 공식 기록이 된 시점이다. | 승인 시 필수 | 시스템 | 시스템 | 공개 가능 | Timeline |

### 유형 후보

- 문제 발견
- 문제 정의 수정
- 가설 생성
- 가설 반박
- 실험
- 사용자 피드백
- 기능 추가
- 기능 제거
- 판단 유지
- 판단 수정
- 방향 전환
- 릴리즈
- 인수인계 메모

### 상태 필드 후보

작업 상태 후보:

- 초안
- 수정 중
- 승인됨
- 보류됨

공개 상태 후보:

- 내부 전용
- 공개 가능
- 공개됨
- 민감 정보 포함

### 주요 관계

- Project에 속한다.
- Builder가 작성하거나 승인한다.
- Rough Note 또는 AI Structured Draft에서 전환될 수 있다.
- Problem Definition, Hypothesis, Feedback과 선택적으로 연결될 수 있다.

### 공개 가능성

승인된 카드라도 공개 상태가 공개됨이 아니면 외부에 노출하지 않는다. 민감 정보 포함 상태는 공개 전 확인이 필요하다.

### 1차 구현 포함 여부

1차 필수다.

### 추가 검토 필요

- 원문 메모를 참조만 할지, 카드에 원문 스냅샷을 함께 저장할지 검토가 필요하다.
- 연결 대상이 여러 개인 경우 연결 테이블이 필요할 수 있지만 1차에서는 직접 선택 참조를 우선 검토한다.
