# 공개 필드와 상태 필드 후보

> 이 문서는 DB 스키마 초안 문서다. 실제 SQL, Supabase migration, RLS 정책, API, 프론트엔드 구현은 작성하지 않는다. 테이블명과 필드명은 후보이며 최종 확정이 아니다.

## 1. 상태값을 분리해야 하는 이유

BuildMap에서는 작업 상태, 공개 상태, 검토 상태, 진행 상태가 서로 다른 의미를 가진다. 이를 하나의 상태 필드에 섞으면 다음 문제가 생긴다.

- 승인되었지만 비공개인 Change Card를 표현하기 어렵다.
- 운영 중이지만 비공개인 Project를 표현하기 어렵다.
- AI Draft의 전환 상태와 Change Card의 공식 승인 상태가 충돌한다.
- Feedback의 검토 상태와 공개 여부가 섞인다.

따라서 6단계 스키마 초안에서는 상태축 분리 원칙을 확정한다.

## 2. 상태 필드 후보 요약

| 대상 | 상태축 | 상태 후보 | 누가 변경하는가 | 영향 화면 | 1차 필요 여부 |
|---|---|---|---|---|---|
| Change Card | 작업 상태 | 초안, 수정 중, 승인됨, 보류됨 | Builder, 시스템 | 카드 검토, Timeline | 필수 |
| Change Card | 공개 상태 | 내부 전용, 공개 가능, 공개됨, 민감 정보 포함 | Builder | 공개 페이지 | 필수 |
| Project | 진행 상태 | 아이디어, 제작 중, 테스트 중, 베타, 운영 중, 일시 중단, 종료됨 | Builder | 대시보드, 공개 페이지 | 필수 |
| Project | 공개 상태 | 비공개, 링크 공개, 전체 공개 | Builder | 공개 페이지, 권한 | 필수 |
| AI Draft | 전환 상태 | 생성 중, 생성됨, 수정 중, Change Card로 전환됨, 보류됨, 실패 | 시스템, Builder | AI 초안 검토 | 필수 |
| Feedback | 검토 상태 | 새 피드백, 검토 중, 반영됨, 반영하지 않음 | Builder | 피드백 검토 | 필수 후보 |
| Feedback | 공개 상태 | 내부 검토, 공개 선택됨 | Builder | 공개 페이지 | 필수 후보 |
| Hypothesis | 검증 상태 | 가정, 검증 중, 일부 확인, 확인됨, 반박됨, 보류 | Builder | 문제/가설, 공개 페이지 | 필수 |
| Feedback Request | 요청 상태 | 열림, 닫힘, 보류 후보 | Builder | 피드백 요청 | 추가 검토 |

## 3. Change Card 작업 상태

- 초안: 아직 공식 기록이 아니다.
- 수정 중: Builder가 AI 초안 또는 카드 내용을 조정 중이다.
- 승인됨: 공식 Change Card로 인정되어 Timeline 표현 대상이 된다.
- 보류됨: 아직 공식 기록으로 반영하지 않기로 한 상태다.

주의: 승인됨은 공개됨이 아니다. 공개 여부는 공개 상태가 판단한다.

## 4. Change Card 공개 상태

- 내부 전용: 외부에 노출하지 않는다.
- 공개 가능: 공개할 수 있지만 아직 공개되지는 않았다.
- 공개됨: 공개 프로젝트 페이지에 노출될 수 있다.
- 민감 정보 포함: 공개 전 확인이 필요하다.

## 5. Project 진행 상태

- 아이디어: 문제의식 또는 아이디어를 기록하기 시작한 상태
- 제작 중: 실제 제작 또는 구체화가 진행 중인 상태
- 테스트 중: 사용자 테스트나 검증을 진행 중인 상태
- 베타: 제한적 공개 또는 초기 사용자 테스트가 가능한 상태
- 운영 중: 외부 사용 또는 운영이 진행 중인 상태
- 일시 중단: 진행이 멈춘 상태
- 종료됨: 더 이상 진행하지 않는 상태

## 6. Project 공개 상태

- 비공개: Builder 내부 기록만 가능하다.
- 링크 공개: 링크를 아는 사람만 볼 수 있다.
- 전체 공개: 탐색 화면과 공개 페이지에서 노출될 수 있다.

Project가 운영 중이어도 비공개일 수 있고, 테스트 중이어도 링크 공개일 수 있다.

## 7. AI Draft 상태

- 생성 중: AI 구조화가 진행 중이다.
- 생성됨: 초안이 생성되었다.
- 수정 중: Builder가 초안을 수정 중이다.
- Change Card로 전환됨: 초안이 공식 카드 후보로 넘어갔다.
- 보류됨: 초안을 사용하지 않기로 했다.
- 실패: AI 구조화가 실패했다.

AI Draft에는 승인됨 상태를 두지 않는다.

## 8. Feedback 상태

검토 상태:

- 새 피드백
- 검토 중
- 반영됨
- 반영하지 않음

공개 상태:

- 내부 검토
- 공개 선택됨

Feedback 내용은 기본 내부 검토이고, Builder가 선택한 것만 공개할 수 있다.

## 9. 추가 검토 필요 사항

- 상태 변경 이력을 별도 저장할지
- 상태 후보를 DB enum으로 둘지 문자열 후보로 둘지
- 민감 정보 포함 상태를 누가 지정할지
- 공개 가능과 공개됨 사이에 검토 단계가 필요한지
