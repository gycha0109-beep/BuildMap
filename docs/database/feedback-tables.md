# Feedback Request와 Feedback 테이블 후보

> 이 문서는 DB 스키마 초안 문서다. 실제 SQL, Supabase migration, RLS 정책, API, 프론트엔드 구현은 작성하지 않는다. 테이블명과 필드명은 후보이며 최종 확정이 아니다.

## 1. 포함할 테이블 후보

- `feedback_requests`
- `feedbacks`

Feedback은 일반 댓글이 아니다. 특정 판단, 질문, 검증 요청에 연결되는 근거다.

---

## 2. feedback_requests

### 제품상 역할

Feedback Request는 Builder가 특정 판단에 대해 피드백을 요청하는 단위다. 공개 프로젝트 전체에 막연한 의견을 받는 것이 아니라, 문제 정의, 가설, Change Card, 공개 페이지 등 특정 대상에 질문을 붙이는 구조다.

### 1차 포함 여부

1차 필수다.

### 연결 대상 후보

- Project
- Problem Definition
- Hypothesis
- Change Card
- Public Project Page 표현

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| Project 참조 | 어떤 프로젝트의 요청인지 연결한다. | 필수 | Builder/시스템 | 제한적 수정 | 공개 가능 | 피드백 요청 |
| 생성 Builder 참조 | 요청한 Builder를 연결한다. | 필수 | 시스템 | 시스템 | 일부 공개 가능 | 공개 페이지 |
| 연결 대상 유형 | 프로젝트/문제/가설/카드 등 대상 성격이다. | 필수 후보 | Builder | Builder | 공개 가능 | 피드백 요청 |
| 연결 대상 참조 | 구체 대상 후보를 연결한다. | 선택/강력 권장 | Builder | Builder | 공개 가능 | 피드백 요청 |

| 공개 상태 | 요청 노출 여부다. | 필수 후보 | Builder | Builder | 공개 판단 | 공개 페이지 |
| 요청 상태 | 열림, 닫힘, 보류 등 후보 | 필수 후보 | Builder | Builder | 공개 가능 | 피드백 요청 |
| 생성 시점 | 요청 생성 시점이다. | 필수 후보 | 시스템 | 시스템 | 공개 가능 | 공개 페이지 |

### 추가 검토 필요

Feedback Request 상태 후보는 아직 최종 확정하지 않는다. 1차에서는 열림/닫힘 정도로 충분할 수 있다.

## 3. feedbacks

### 제품상 역할

Feedback은 로그인 사용자 또는 최소 계정 사용자가 특정 Feedback Request 또는 판단 대상에 남기는 근거다. 1차에서 비로그인 피드백은 보류한다.

### 1차 포함 여부

1차 필수다.

### 주요 필드 후보

| 필드 후보 | 의미 | 필수 여부 | 생성 주체 | 수정 주체 | 공개 가능성 | 관련 화면 |
|---|---|---|---|---|---|---|
| Feedback Request 참조 | 어떤 요청에 대한 피드백인지 연결한다. | 필수 후보 | 시스템 | 시스템 | 공개 여부 연동 | 피드백 작성 |
| 작성 사용자 참조 | 누가 작성했는지 연결한다. | 필수 | 시스템 | 제한적 수정 | 선택 공개 | 피드백 작성 |
| Project 참조 | 프로젝트 단위 조회를 돕는다. | 필수 후보 | 시스템 | 시스템 | 내부 기본 | Builder 검토 |
| 피드백 유형 | 공감, 이해 안 됨, 개선 제안 등 | 필수 후보 | 사용자 | 사용자 | 선택 공개 | 피드백 작성 |
| 피드백 내용 | 실제 응답 내용이다. | 필수 | 사용자 | 사용자/제한 | 내부 기본 | Builder 검토 |
| 테스터 참여 의사 | 테스트 참여 여부 후보 | 선택 | 사용자 | 사용자 | 내부 기본 | 피드백 작성 |
| 검토 상태 | 새 피드백, 검토 중, 반영됨, 반영하지 않음 | 필수 후보 | 시스템/Builder | Builder | 내부 기본 | Builder 검토 |
| 공개 상태 | 내부 검토, 공개 선택됨 | 필수 후보 | Builder | Builder | 공개 판단 | 공개 페이지 |
| 관련 Change Card 후보 | 반영된 경우 연결한다. | 선택 | Builder/시스템 | Builder | 선택 공개 | 카드 작성 |
| 생성 시점 | 피드백 작성 시점이다. | 필수 후보 | 시스템 | 시스템 | 선택 공개 | Builder 검토 |

### Feedback 검토 상태 후보

- 새 피드백
- 검토 중
- 반영됨
- 반영하지 않음

### Feedback 공개 상태 후보

- 내부 검토
- 공개 선택됨

### 공개 정책

- Feedback Request는 공개 가능하다.
- Feedback 내용은 기본적으로 Builder 내부 검토용이다.
- Builder가 선택한 Feedback만 공개할 수 있다.
- Feedback이 반영되면 새 Change Card로 이어질 수 있다.
- Feedback을 일반 댓글창처럼 노출하지 않는다.

## 4. 추가 검토 필요 사항

- Feedback이 Feedback Request 없이 특정 Change Card에 직접 달릴 수 있는지
- 공개 선택된 Feedback의 작성자 표시 범위
- 부적절한 피드백 신고/숨김 정책
- 테스터 신청을 Feedback 내부 필드로 둘지 별도 테이블로 뺄지
