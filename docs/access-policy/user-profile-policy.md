# 사용자 프로필 접근 정책

> 본 문서는 BuildMap 7단계 문서다. 7단계는 실제 RLS SQL 작성 단계가 아니라, SQL 작성 전 권한/공개/접근 정책을 자연어로 고정하는 단계다. `docs/decisions/phase6-5-db-schema-corrections.md`를 최우선 기준으로 삼는다.

## 1. 문서 목적

이 문서는 user profile 후보, Builder Profile, Scout Profile 후보의 접근 정책을 정리한다. 핵심은 공개 프로필 정보와 내부 계정 정보를 분리하는 것이다.

## 2. user profile 후보 정책

### 읽기 정책

- 로그인 사용자는 자신의 user profile 후보를 읽을 수 있다.
- 다른 사용자는 공개로 설정된 표시 정보만 읽을 수 있다.
- 이메일, 인증 식별자, 내부 계정 상태는 외부 사용자에게 노출하지 않는다.

### 수정 정책

- 로그인 사용자는 자신의 표시명, 기본 소개, 프로필 이미지 후보를 수정할 수 있다.
- 인증 식별자와 내부 계정 상태는 일반 사용자가 직접 수정하지 않는다.

### 공개 가능 정보

- 표시명
- 공개 프로필 이미지 후보
- 사용자가 공개로 둔 간단한 소개

### 비공개 정보

- 이메일
- 인증 ID
- 계정 상태
- 약관 동의 상세
- 내부 운영 메모 후보

## 3. Builder Profile 정책

### 읽기 정책

- Builder 본인은 자신의 Builder Profile 전체를 읽을 수 있다.
- 공개 프로젝트 페이지에서는 Builder가 공개로 설정한 정보만 노출한다.
- 비공개 프로젝트와 연결된 Builder 내부 정보는 외부에 노출하지 않는다.

### 수정 정책

- Builder Profile은 해당 Builder 본인이 수정할 수 있다.
- 1차에서는 관리자 대리 수정 정책을 다루지 않는다.

### 공개 페이지에서 노출 가능한 Builder 정보

- 공개 이름
- 공개 소개
- 공개 역할 태그
- 공개 프로젝트 목록 후보

### 노출하지 않는 정보

- 이메일
- 인증 식별자
- 내부 계정 상태
- 비공개 프로젝트 목록
- 내부 메모

## 4. Scout Profile 후보 정책

Scout Profile은 1차 선택 데이터다.

### 읽기 정책

- Scout 성격의 로그인 사용자는 자신의 Scout Profile 후보를 읽을 수 있다.
- 다른 사용자가 Scout Profile을 읽는 정책은 후순위다.

### 수정 정책

- 해당 로그인 사용자가 자신의 관심 분야, 탐색 목적 등을 수정할 수 있다.

### 1차 보류

- Scout 공개 프로필
- 채용 담당자/헤드헌터 전용 프로필
- 투자자 인증 정보
- Scout 평판 또는 점수

## 5. Feedback 작성 시 표시 정보

Feedback 작성 시 외부에 노출 가능한 작성자 정보는 제한한다.

- 기본적으로 Feedback 내용은 Builder 내부 검토용이다.
- Builder가 Feedback을 공개 선택하더라도 작성자의 이메일과 인증 식별자는 노출하지 않는다.
- 공개 표시명 또는 제한된 닉네임 후보만 노출한다.
- 익명 공개 여부는 추가 검토 필요다.

## 6. 정책 예시

- 로그인 사용자는 자신의 user profile을 읽고 수정할 수 있다.
- 공개 프로젝트 페이지에서는 Builder가 공개로 설정한 Builder Profile 정보만 노출한다.
- 이메일과 인증 식별자는 공개 페이지에 노출하지 않는다.
- Feedback 작성자의 개인정보는 기본적으로 외부에 노출하지 않는다.

## 7. 후순위 검토 사항

- 프로필 공개 범위 세부 설정
- Scout 공개 프로필
- 관리자에 의한 프로필 제재 표시
- 신고 누적 사용자 처리
- 조직/팀 소속 표시
