# 공개/비공개/민감도 모델

> 본 문서는 BuildMap 7단계 문서다. 7단계는 실제 RLS SQL 작성 단계가 아니라, SQL 작성 전 권한/공개/접근 정책을 자연어로 고정하는 단계다. `docs/decisions/phase6-5-db-schema-corrections.md`를 최우선 기준으로 삼는다.

## 1. 문서 목적

이 문서는 BuildMap의 공개 상태, 민감도, 공개 페이지 접근 상태를 정리한다. 6.5단계 결정에 따라 공개 상태와 민감도는 분리한다.

## 2. Project 공개 상태 후보

| 상태 | 의미 | 외부 노출 | 설정 주체 | RLS 설계 고려 |
|---|---|---|---|---|
| 비공개 | Project Owner 내부 작업용 | 외부 노출 없음 | Project Owner | 외부 방문자 차단 |
| 링크 공개 | 식별자를 가진 사용자가 공개 정보 접근 | 제한 공개 | Project Owner | share token 등 접근 조건 필요 |
| 전체 공개 | 공개 탐색/검색/공개 URL 접근 가능 | 공개 정보 노출 | Project Owner | 누구나 읽기 가능한 정보 제한 필요 |

Project가 비공개이면 Change Card가 `공개됨` 상태라도 외부에 노출하지 않는다.

## 3. Change Card 공개 상태 후보

| 상태 | 의미 | 외부 노출 | 설정 주체 | 주의점 |
|---|---|---|---|---|
| 내부 전용 | 내부 판단 기록 | 외부 노출 없음 | Project Owner/Builder | 공개 Timeline 제외 |
| 공개 가능 | 공개 후보로 검토 가능 | 아직 외부 노출 아님 | Project Owner/Builder | 공개됨과 혼동 금지 |
| 공개됨 | 외부 공개 후보 | 조건 만족 시 노출 | Project Owner | Project 공개 상태와 민감도도 함께 확인 |

`공개 가능`은 공개된 상태가 아니다.

## 4. Change Card 민감도 후보

| 상태 | 의미 | 외부 노출 | 설정 주체 | RLS 설계 고려 |
|---|---|---|---|---|
| 일반 | 공개 검토에 큰 제한이 없는 기록 | 공개 조건 충족 시 가능 | Builder | 공개 Timeline 후보 |
| 민감 정보 포함 | 개인정보, 내부 전략, 보안 정보 등이 포함된 기록 | 기본적으로 공개 차단 또는 재검토 | Builder/관리자 후보 | 공개 상태와 별도 축으로 차단 조건 필요 |

민감 정보 포함은 공개 상태가 아니라 민감도 플래그다.

## 5. Feedback 공개 상태 후보

| 상태 | 의미 | 외부 노출 | 설정 주체 |
|---|---|---|---|
| 내부 검토 | Builder 내부 검토용 Feedback | 외부 노출 없음 | 기본 상태 |
| 공개 선택됨 | Builder가 외부 노출을 선택한 Feedback | 제한된 표시 정보와 함께 공개 가능 | Project Owner |

Feedback 내용은 기본 공개가 아니다.

## 6. Feedback Request 공개 상태 후보

| 상태 | 의미 | 외부 노출 |
|---|---|---|
| 내부 요청 | Builder 내부 확인용 요청 | 외부 노출 없음 |
| 공개 요청 | 공개 프로젝트 방문자 또는 로그인 사용자에게 피드백 요청 | 공개 가능 |

Feedback Request는 공개될 수 있지만, Feedback 내용은 별도 공개 정책을 따른다.

## 7. Public Project Page 접근 상태

| 상태 | 의미 |
|---|---|
| 접근 불가 | 비공개 Project이거나 링크 조건 불충족 |
| 링크 접근 가능 | 링크 공개 Project에서 유효한 접근 식별자 조건 충족 |
| 전체 공개 접근 가능 | 전체 공개 Project의 공개 페이지 접근 |

## 8. 반드시 지켜야 할 원칙

- `공개 가능`은 `공개됨`이 아니다.
- 민감 정보 포함은 공개 상태가 아니라 민감도 플래그다.
- 링크 공개와 전체 공개는 다르다.
- 비공개 Project의 Change Card는 외부에 노출되지 않는다.
- Project가 전체 공개라도 내부 전용 Change Card와 민감 정보 포함 Change Card는 공개 Timeline에 노출되지 않는다.
- 공개 Timeline은 Project 공개 상태, Change Card 작업 상태, Change Card 공개 상태, 민감도를 함께 고려한다.
