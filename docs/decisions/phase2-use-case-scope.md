# 2단계 유즈케이스 범위 결정

## 문서 목적

이 문서는 BuildMap 2단계에서 확정한 것과 보류한 것을 기록한다.

이번 단계의 목적은 구현이 아니라 핵심 유즈케이스 문서화다. 화면, DB, API, 권한, 결제, 추천 알고리즘은 확정하지 않는다.

## 1. 확정한 것

### 1. 2단계의 중심은 Builder의 판단 흐름 기록이다

BuildMap의 중심은 Scout 탐색, 채용, 평가, 히트맵이 아니다.

2단계의 중심은 Builder가 프로젝트 진행 중 발생한 문제, 가설, 실험, 피드백, 판단, 방향 전환을 기록하는 흐름이다.

### 2. 변화 카드는 의사결정 단위다

변화 카드는 단순 작업 로그가 아니다. 변화 카드는 프로젝트에서 발생한 하나의 의사결정 단위다.

변화 카드에는 문제 발견, 가설 생성, 실험, 사용자 피드백, 기능 추가, 기능 제거, 판단 유지, 판단 수정, 방향 전환, 릴리즈, 인수인계 메모 등이 포함될 수 있다.

### 3. AI는 변화 카드 초안을 구조화한다

AI는 프로젝트를 대신 만들어주거나 과장된 포트폴리오 문장을 작성하는 역할이 아니다.

AI는 Builder의 거친 메모, 피드백, 실험 결과, 회의 내용을 문제·가설·근거·판단·변경·다음 확인 사항으로 구조화한다.

### 4. Builder가 최종 승인자다

AI가 만든 변화 카드 초안은 공식 기록이 아니다.

Builder가 수정하고 승인해야 Decision Timeline에 반영된다.

### 5. 공개 프로젝트 페이지는 필요하지만 내부 기록 후 공개 전환 구조가 적합하다

모든 기록을 처음부터 공개하도록 강제하지 않는다.

기본 흐름은 내부 기록 → 정리 → 공개 전환이다. Builder는 공개 가능한 변화 카드만 외부에 보여줄 수 있어야 한다.

### 6. Scout는 초기에는 피드백 제공자, 테스터, 협업자, 발견자 역할이 우선이다

Scout는 단순 채용 담당자가 아니다.

초기 Scout는 프로젝트를 발견하고, 피드백을 주고, 테스터가 되고, 협업자로 접근하고, 멘토링하거나 팀원을 탐색하는 사람이다.

### 7. 채용/헤드헌팅은 장기 확장이다

채용 담당자와 헤드헌터는 장기 확장 사용자로 다룬다.

초기 문서에서 BuildMap을 채용 플랫폼처럼 보이게 하지 않는다. Scout가 보는 정보는 점수화된 평가가 아니라 프로젝트 기록에서 드러나는 근거 중심 정보다.

### 8. 프로젝트 인수인계는 중요한 유즈케이스 후보로 포함한다

프로젝트 인수인계는 BuildMap의 차별점을 강화한다.

새 팀원, 협업자, 멘토, 인수자는 Decision Timeline을 통해 프로젝트가 왜 현재 모습이 되었는지 빠르게 이해할 수 있어야 한다.

다만 인수인계 전용 복잡한 기능은 이번 단계에서 구현 범위로 확정하지 않는다.

## 2. 보류한 것

이번 단계에서는 다음을 보류한다.

- 구체 화면 UI
- DB 구조
- 변화 카드 필드의 최종 스키마
- AI 입력 방식
- AI 자동화 수준
- Scout 권한
- 알림 방식
- 히트맵 산식
- 공개/비공개 권한 세부 정책
- 채용 기능
- Project DNA 표현
- 역량 점수화

## 3. 보류 이유

현재 단계에서 위 항목을 확정하면 BuildMap의 핵심 정체성이 흐려질 수 있다.

BuildMap은 먼저 다음을 분명히 해야 한다.

> 프로젝트의 현재 버전이 아니라, 현재 버전에 도달한 판단 흐름을 어떻게 기록하고 공유할 것인가?

세부 구현과 권한, 화면, DB, 평가 체계는 이 질문에 대한 답이 충분히 안정된 뒤 다루는 것이 맞다.

## 4. 다음 단계로 넘길 질문

3단계로 넘어가기 전 다음 질문을 확인해야 한다.

1. 첫 구현에서 Builder 핵심 흐름을 어디까지 포함할 것인가?
2. 변화 카드의 최소 필드는 무엇이어야 하는가?
3. AI 구조화 초안은 어떤 입력에서 시작하는가?
4. 공개 프로젝트 페이지는 어떤 정보 순서로 구성되는가?
5. 내부 기록과 공개 전환의 기본 정책은 어떻게 둘 것인가?
6. Scout 탐색은 어느 수준까지 초기 흐름에 포함할 것인가?
7. 인수인계는 별도 모드로 둘 것인가, 공개 프로젝트 페이지의 확장으로 둘 것인가?
