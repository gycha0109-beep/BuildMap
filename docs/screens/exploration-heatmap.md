# 프로젝트 히트맵 탐색 화면

## 1. 화면 목적

프로젝트 히트맵은 BuildMap에 공개된 프로젝트를 한눈에 발견하는 탐색 화면이다.

히트맵은 인기 사이트 순위가 아니다. 히트맵은 지금 살아 움직이는 프로젝트를 발견하는 입구다.

BuildMap에서 중요한 것은 완성된 결과물의 크기가 아니라 프로젝트가 어떤 판단을 쌓고, 어떤 피드백을 받고, 어떤 전환을 만들고 있는지다.

## 2. 왜 일반 리스트보다 히트맵이 적합할 수 있는가

일반 리스트는 최신순, 인기순, 추천순으로 프로젝트를 나열한다. 그러나 BuildMap은 단순 결과물 목록이 아니라 프로젝트의 변화와 활동성을 보여줘야 한다.

히트맵은 다음 장점이 있다.

- 진행 중인 프로젝트의 활동성을 한눈에 보여줄 수 있다.
- 피드백 요청, 테스터 모집, 방향 전환 같은 현재성을 드러낼 수 있다.
- 사용자가 “지금 참여할 수 있는 프로젝트”를 빠르게 발견할 수 있다.
- 단순 조회수보다 내부 활동 지표를 강조할 수 있다.

단, 히트맵 산식은 아직 확정하지 않는다.

## 3. 타일이 의미하는 것

히트맵의 각 타일은 하나의 공개 프로젝트를 의미한다.

타일에는 다음 정보가 표시될 수 있다.

- 프로젝트명
- 한 줄 정의
- 현재 상태
- 현재 필요한 것
- 최근 전환
- 최근 변화 카드 수
- 피드백 요청 여부
- 테스터 모집 여부
- 협업자 필요 여부

타일은 프로젝트의 완성도를 보여주는 것이 아니라 현재 활동성과 참여 가능성을 보여준다.

## 4. 타일 크기의 가능한 기준

타일 크기는 프로젝트의 상대적 활동성을 표현할 수 있다.

가능한 기준은 다음이다.

- 최근 변화 카드 수
- 최근 공개 업데이트 수
- 최근 피드백 수
- 테스터 신청 수
- 팔로우 수
- 저장 수
- 최근 핵심 전환 발생 여부

초기에는 산식을 확정하지 않는다. 외부 트래픽 데이터는 초기 기준에서 제외한다.

### 추가 검토 필요

- 타일 크기를 단일 지표로 할지 복합 지표로 할지
- 최근 기간을 7일, 30일, 전체 중 무엇으로 볼지
- 카드 수가 많은 프로젝트가 과도하게 유리해지는 문제

## 5. 타일 색상의 가능한 기준

타일 색상은 프로젝트의 상태나 현재 필요한 것을 표현할 수 있다.

가능한 기준은 다음이다.

- 피드백 요청 중
- 테스터 모집 중
- 방향 전환 발생
- 최근 공개됨
- 협업자 필요
- 검증 중
- 보류 중

색상 기준은 디자인 시스템에서 확정한다. 이번 단계에서는 제품 의미만 정리한다.

### 추가 검토 필요

- 색상이 활동성을 의미할지 상태를 의미할지
- 접근성 관점에서 색상 외 보조 표시가 필요한지
- 너무 많은 색상이 의미 혼란을 만들지

## 6. 타일 배지 예시

타일에는 다음 배지가 표시될 수 있다.

- 테스터 모집
- 피드백 요청
- 급상승
- 첫 공개
- 방향 전환
- 베타
- 협업자 필요

배지는 프로젝트의 현재 참여 지점을 보여주기 위한 장치다. 단순 마케팅 배지가 아니어야 한다.

## 7. 타일 클릭 후 이동 경로

타일을 클릭하면 공개 프로젝트 페이지로 이동한다.

기본 이동 경로:

```text
히트맵 타일
→ 공개 프로젝트 페이지
→ Decision Timeline
→ 특정 변화 카드
→ 피드백 작성 또는 테스터 신청
```

Scout는 타일에서 프로젝트를 빠르게 발견하고, 공개 프로젝트 페이지에서 맥락을 이해하고, Timeline에서 판단 근거를 확인한다.

## 8. 내부 활동 지표 기반 원칙

초기 히트맵은 외부 트래픽 데이터가 아니라 BuildMap 내부 활동 지표를 기반으로 가정한다.

가능한 내부 지표:

- 최근 변화 카드 수
- 공개 변화 카드 수
- 피드백 수
- 테스터 신청 수
- 팔로우 수
- 저장 수
- 공개 업데이트 수

외부 조회수, 외부 SNS 공유 수, 외부 사이트 트래픽은 초기 제외한다.

## 9. 초기 구현에서는 카드 그리드로 대체 가능

히트맵은 탐색 아이디어로 중요하지만, 초기 구현에서 반드시 복잡한 시각화를 만들 필요는 없다.

초기에는 다음 방식으로 대체 가능하다.

- 프로젝트 카드 그리드
- 최근 활동순 정렬
- 피드백 요청 필터
- 테스터 모집 필터
- 최근 전환 배지

이 방식으로도 “지금 살아 움직이는 프로젝트를 발견하는 입구”라는 목적을 유지할 수 있다.

## 10. 초기 구현 범위

초기 구현 후보는 다음이다.

- 공개 프로젝트 카드 그리드
- 프로젝트 상태 표시
- 현재 필요한 것 표시
- 피드백 요청 배지
- 테스터 모집 배지
- 최근 활동 필터
- 프로젝트 분야 필터

## 11. 후순위 요소

- 실제 히트맵 시각화
- 타일 크기/색상 산식
- 급상승 프로젝트
- 개인화 탐색
- 고급 활동 지표
- 외부 트래픽 연동

## 12. 추가 검토 필요

- 히트맵 산식
- 타일 크기 기준
- 타일 색상 기준
- 활동성 기간 기준
- 조작 가능성 방지
- 단순 인기 순위로 오해되지 않게 하는 문구
