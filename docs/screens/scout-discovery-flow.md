# Scout 탐색 화면과 흐름

## 1. 문서 목적

이 문서는 Scout가 BuildMap에서 프로젝트와 Builder를 발견하는 화면 흐름을 정리한다.

Scout는 초기에는 채용 담당자가 아니다. 초기 Scout는 프로젝트 발견자, 피드백 제공자, 테스터, 협업자, 멘토, 팀원 탐색자에 가깝다.

Scout 화면의 핵심은 사람 평가가 아니라 프로젝트 맥락 이해와 연결이다.

## 2. 전체 흐름

```text
Scout 역할 선택
→ 관심 분야 선택
→ 프로젝트 탐색
→ 공개 프로젝트 페이지 확인
→ Decision Timeline 확인
→ 특정 변화 카드에 피드백 작성
→ 테스터 신청
→ 프로젝트 저장
→ Builder에게 맥락 기반 연락
```

초기 구현에서는 공개 프로젝트 탐색, 피드백 작성, 테스터 신청, 프로젝트 저장 정도면 충분하다. 채용/헤드헌팅 기능은 제외한다.

## 3. 단계별 흐름

## 3.1 Scout 역할 선택

### 사용자의 상황

사용자는 프로젝트나 Builder를 발견하기 위해 BuildMap을 사용한다.

### 화면에서 보여야 할 핵심 정보

- Scout는 단순 채용 담당자가 아니라는 설명
- 초기 Scout 유형
  - 테스터
  - 초기 사용자
  - 협업자
  - 멘토
  - 창업자
  - 팀원 탐색자
  - 커뮤니티 운영자
- 장기 Scout 유형은 채용 담당자, 헤드헌터, 투자자까지 확장될 수 있다는 안내

### 핵심 행동

- Scout로 시작하기 선택
- 자신의 탐색 목적 선택

### 다음 화면으로 넘어가는 조건

- 탐색 목적이 선택되거나 건너뛰기 선택

### 후순위 요소

- Scout 프로필
- 신뢰도 관리
- 조직 계정

## 3.2 관심 분야 선택

### 사용자의 상황

Scout는 관심 있는 프로젝트 분야를 좁히고 싶다.

### 화면에서 보여야 할 핵심 정보

- 분야 선택은 추천과 필터를 위한 메타데이터라는 안내
- 프로젝트 분야 예시
  - AI
  - 생산성
  - 교육
  - 커뮤니티
  - 개발도구
  - 뷰티/헬스
  - 로컬 서비스
  - 콘텐츠
  - 기타

### 핵심 행동

- 관심 분야 선택
- 필요한 도움 유형 선택 가능

### 다음 화면으로 넘어가는 조건

- 프로젝트 탐색 화면으로 이동

### 후순위 요소

- 관심 분야별 알림
- 저장된 검색 조건

## 3.3 프로젝트 탐색

### 사용자의 상황

Scout는 완성된 제품이 아니라 진행 중인 프로젝트를 발견한다.

### 화면에서 보여야 할 핵심 정보

- 프로젝트명
- 한 줄 정의
- 현재 상태
- 현재 필요한 것
- 최근 전환
- 최근 활동
- 피드백 요청 여부
- 테스터 모집 여부

### 핵심 행동

- 프로젝트 카드 클릭
- 필터 적용
- 저장
- 피드백 요청이 있는 프로젝트 확인

### 다음 화면으로 넘어가는 조건

- 공개 프로젝트 페이지로 이동

### 초기 Scout 탐색 필터

- 프로젝트 분야
- 프로젝트 단계
- 필요한 도움
- 최근 활동
- 변화 카드 유형
- 테스터 모집 여부
- 협업자 모집 여부

### 후순위 요소

- 고급 검색
- Scout별 추천
- 저장 목록

## 3.4 공개 프로젝트 페이지 확인

### 사용자의 상황

Scout는 프로젝트의 결과물보다 판단 흐름을 확인하려 한다.

### 화면에서 보여야 할 핵심 정보

- 문제 정의
- 현재 가설
- 최근 전환
- 현재 필요한 것
- 프로젝트 진화도
- 공개 변화 카드
- 피드백 요청

### 핵심 행동

- 프로젝트 맥락 확인
- Decision Timeline으로 이동
- 피드백 작성
- 테스터 신청
- 저장

### 다음 화면으로 넘어가는 조건

- 특정 변화 카드 또는 피드백 요청을 선택한다.

### 후순위 요소

- 프로젝트 비교
- Scout 메모
- 외부 공유

## 3.5 Decision Timeline 확인

### 사용자의 상황

Scout는 이 프로젝트가 어떤 판단을 거쳐 현재 상태가 되었는지 알고 싶다.

### 화면에서 보여야 할 핵심 정보

- 핵심 전환점
- 문제 정의 변경
- 가설 반박
- 사용자 피드백 반영
- 기능 제거 판단
- 릴리즈 목적

### 핵심 행동

- 카드 유형 필터
- 핵심 전환만 보기
- 특정 변화 카드 열기

### 다음 화면으로 넘어가는 조건

- 변화 카드 피드백 작성 또는 프로젝트 저장

### 후순위 요소

- Scout용 요약 보기
- 인수인계 관점 요약

## 3.6 특정 변화 카드에 피드백 작성

### 사용자의 상황

Scout가 프로젝트 전체 평가가 아니라 특정 판단에 대해 의견을 남긴다.

### 화면에서 보여야 할 핵심 정보

- 변화 카드의 문제/근거/판단/다음 확인 사항
- Builder가 요청한 질문
- 피드백 유형
  - 공감
  - 이해 안 됨
  - 개선 제안
  - 사용 경험 공유
  - 테스트 참여 의사

### 핵심 행동

- 피드백 작성
- 피드백 유형 선택
- 공개 여부 선택 가능

### 다음 화면으로 넘어가는 조건

- 피드백 제출 후 공개 프로젝트 페이지 또는 Timeline으로 돌아간다.

### 후순위 요소

- 피드백 신뢰도
- 피드백 스레드
- Builder 답변

## 3.7 테스터 신청

### 사용자의 상황

Scout가 프로젝트의 테스터 모집 요청을 보고 참여하려 한다.

### 화면에서 보여야 할 핵심 정보

- 테스트 목적
- 테스트 대상
- 예상 소요 시간
- Builder가 확인하려는 질문
- 개인정보 또는 연락 방식 안내

### 핵심 행동

- 테스터 신청
- 간단한 참여 가능 정보 입력

### 다음 화면으로 넘어가는 조건

- 신청 완료 후 Builder에게 전달된다.

### 후순위 요소

- 일정 조율
- 테스터 관리
- 테스트 결과 입력 양식

## 3.8 프로젝트 저장

### 사용자의 상황

Scout는 프로젝트를 나중에 다시 보고 싶다.

### 화면에서 보여야 할 핵심 정보

- 저장한 프로젝트 목록으로 이동 가능
- 새 공개 업데이트를 받을 수 있다는 안내

### 핵심 행동

- 저장
- 저장 취소

### 다음 화면으로 넘어가는 조건

- 저장 목록 또는 탐색 화면 유지

### 후순위 요소

- 저장 목록 분류
- 알림 설정
- Scout 메모

## 3.9 Builder에게 맥락 기반 연락

### 사용자의 상황

Scout가 프로젝트 판단 흐름을 보고 협업, 피드백, 테스트 참여를 제안하려 한다.

### 화면에서 보여야 할 핵심 정보

- 연락은 프로젝트 맥락 기반이어야 한다는 안내
- 연결 가능한 변화 카드 또는 피드백 요청
- 메시지 예시

예시:

> 첫 화면을 줄인 판단이 좋았습니다. UX 테스트를 도와보고 싶습니다.

### 핵심 행동

- 맥락 기반 메시지 작성
- 연결된 변화 카드 선택 가능

### 다음 화면으로 넘어가는 조건

- 메시지 전송 또는 외부 연락 링크 이동

### 후순위 요소

- 내부 메시징
- 협업 제안 관리
- 팀원 초대 흐름

## 4. 초기 Scout 탐색에서 제외할 것

초기 Scout 탐색에서는 다음을 제외한다.

- 채용 공고 등록
- 헤드헌팅 계약
- 지원자 관리 시스템
- AI 역량 점수
- 자동 랭킹
- 투자 매칭

이 항목들은 BuildMap이 채용 플랫폼처럼 보이게 만들 수 있으므로 장기 확장으로만 남긴다.

## 5. 초기 구현 범위

초기 Scout 흐름은 다음 정도로 제한한다.

- 공개 프로젝트 탐색
- 기본 필터
- 공개 프로젝트 페이지 확인
- Decision Timeline 확인
- 변화 카드 피드백 작성
- 테스터 신청
- 프로젝트 저장

## 6. 추가 검토 필요

- Scout 가입을 필수로 할지
- 피드백 작성에 로그인 필요 여부
- 테스터 신청 정보의 최소 범위
- Builder 연락 방식을 내부 메시징으로 할지 외부 링크로 할지
- Scout 저장 목록의 초기 구현 여부
