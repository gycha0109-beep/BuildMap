# Scout 프로젝트 탐색 화면 텍스트 와이어

## 1. 화면 목적

Scout 프로젝트 탐색 화면은 Scout가 진행 중인 프로젝트를 발견하고, 공개 프로젝트 페이지와 Decision Timeline을 통해 프로젝트 맥락을 이해한 뒤 피드백, 테스트, 저장, 협업으로 이어지는 화면이다.

초기 Scout는 채용 담당자가 아니다. 초기 Scout는 프로젝트 발견자, 피드백 제공자, 테스터, 협업자, 멘토, 팀원 탐색자에 가깝다.

## 2. Scout의 초기 정의

Scout는 다음 역할을 포함한다.

- 프로젝트 발견자
- 피드백 제공자
- 테스터
- 협업자
- 멘토
- 팀원 탐색자

장기적으로 채용 담당자, 헤드헌터, 투자자가 포함될 수 있으나 이번 화면에서는 초기 핵심처럼 다루지 않는다.

## 3. 검색 영역

### 표시 정보

- 검색 입력창
- 관심 분야 선택
- 필요한 도움 중심 검색 안내

### 검색 예시

- AI 플래너
- 테스터 모집
- 문제 정의 피드백
- 사이드프로젝트
- 최근 방향 전환

### 도움말 문구

> 완성된 제품보다 지금 판단이 움직이고 있는 프로젝트를 찾아보세요.

## 4. 필터

### 프로젝트 분야

- AI
- 생산성
- 커뮤니티
- 교육
- 커머스
- 뷰티
- 개발 도구
- 기타

분야 목록은 초기 가정이며 추가 검토 필요.

### 프로젝트 단계

- 아이디어 단계
- 문제 정의 중
- 가설 검증 중
- 초기 실험 중
- 테스트 중
- 운영 중
- 방향 전환 중

### 필요한 도움

- 피드백 필요
- 테스터 모집
- 협업자 필요
- 멘토링 필요
- 사용자 인터뷰 필요

### 최근 활동

- 최근 7일 업데이트
- 최근 30일 업데이트
- 최근 핵심 전환 있음
- 최근 피드백 요청 있음

### 변화 카드 유형

- 문제 정의 수정
- 가설 반박
- 기능 제거
- 방향 전환
- 릴리즈
- 인수인계 메모

### 테스터 모집 여부

- 전체
- 모집 중

### 협업자 모집 여부

- 전체
- 모집 중

## 5. 프로젝트 카드 목록

### 카드에 표시할 정보

- 프로젝트명
- 한 줄 정의
- 현재 필요한 것
- 최근 전환
- 최근 활동
- Builder
- 상태 배지
- 피드백 요청 배지
- 테스터 모집 배지

### 카드 예시

```text
BuildMap
프로젝트의 판단 흐름을 기록하는 Decision Timeline 플랫폼
현재 필요한 것: 공개 페이지 정보 순서 피드백
최근 전환: 히트맵보다 카드 그리드를 먼저 사용하기로 결정
Builder: 지환
[피드백 요청] [최근 전환]
```

## 6. 주요 행동

### 프로젝트 보기

- 공개 프로젝트 페이지로 이동
- 문제 정의, 현재 가설, 최근 전환, 공개 변화 카드 확인

### 피드백 남기기

- 특정 피드백 요청 또는 변화 카드에 연결된 피드백 작성 화면으로 이동

### 테스터 신청

- 프로젝트가 테스터 모집 중인 경우 사용
- 초기에는 단순 참여 의사 표시로 제한 가능

### 저장

- 관심 프로젝트 저장
- 후속 업데이트 확인

### Builder에게 맥락 기반 연락

연락은 단순 DM이 아니라 프로젝트 맥락 기반이어야 한다.

예시:

> 최근 첫 화면 구조를 바꾼 판단을 봤습니다. UX 테스트 관점에서 피드백을 드릴 수 있습니다.

## 7. 제외할 것

이번 화면에서 제외한다.

- 투자 매칭

- 프로젝트 정렬 기준
- 추천 알고리즘을 언제 도입할지
