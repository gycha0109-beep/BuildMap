# 공개 프로젝트 페이지 텍스트 와이어

## 1. 화면 목적

공개 프로젝트 페이지는 제품 소개 페이지가 아니다.

이 페이지는 프로젝트가 어떤 판단을 거쳐 현재 모습이 되었고, 지금 어떤 도움이 필요한지 보여주는 외부 공개 화면이다.

Builder가 공개하기로 선택한 정보만 노출하며, 기본 구조는 **상단 카드형 요약 + 하단 Decision Timeline**이다.

## 2. 주요 사용자

- 외부 사용자
- Scout
- 테스터 후보
- 협업자 후보
- 멘토
- 새 팀원
- 프로젝트에 관심 있는 일반 방문자

## 3. 기본 정보 순서

공개 프로젝트 페이지의 기본 순서는 다음을 유지한다.

```text
프로젝트명
→ 한 줄 정의
→ 현재 상태
→ 현재 필요한 것
→ 문제 정의
→ 현재 가설
→ 최근 전환
→ 프로젝트 진화도
→ 공개 변화 카드
→ 피드백 요청
→ 관련 링크
→ Builder 정보
→ 팔로우 또는 저장
```

## 4. 프로젝트명

### 표시 정보

- 프로젝트명
- 임시 이름 여부는 공개하지 않을 수 있음

### 사용자가 이해해야 하는 것

- 어떤 프로젝트를 보고 있는지

### 주요 행동

- 프로젝트 저장
- 공유

### Builder가 공개 여부를 제어해야 하는 정보

프로젝트명 자체가 민감한 경우 비공개 프로젝트로 유지한다.

### 비개발자 친화적 표현 방식

기술 스택보다 프로젝트 이름과 한 줄 정의를 먼저 보여준다.

### 초기 구현 요소

- 프로젝트명 표시

### 후순위 요소

- 공개 URL 커스터마이징
- 프로젝트 로고

## 5. 한 줄 정의

### 표시 정보

- 프로젝트가 무엇을 하려는지 한 문장으로 표시

### 사용자가 이해해야 하는 것

- 이 프로젝트가 어떤 문제 또는 흐름을 다루는지

### 주요 행동

- 더 읽기
- 현재 필요한 것 확인

### Builder가 공개 여부를 제어해야 하는 정보

공개용 한 줄 정의와 내부용 한 줄 정의를 분리할지 추가 검토 필요.

### 비개발자 친화적 표현 방식

전문 용어보다 문제와 대상 사용자가 이해되는 문장을 우선한다.

### 초기 구현 요소

- 한 줄 정의 표시

### 후순위 요소

- 여러 문구 A/B 히스토리

## 6. 현재 상태

### 표시 정보

- 아이디어 단계
- 문제 정의 중
- 가설 검증 중
- 초기 실험 중
- 테스트 중
- 공개 준비 중
- 운영 중
- 방향 전환 중
- 보류 중

### 사용자가 이해해야 하는 것

- 완성도보다 프로젝트가 어떤 판단 단계에 있는지

### 주요 행동

- 현재 필요한 것 확인
- 최근 전환 확인

### 초기 구현 요소

- 상태 뱃지

### 후순위 요소

- 상태 변경 이력

## 7. 현재 필요한 것

### 표시 정보

- 필요한 피드백
- 테스터 모집 여부
- 협업자 필요 여부
- 멘토링 요청
- 문제 정의 검증 요청

### 사용자가 이해해야 하는 것

- 이 프로젝트에 지금 어떻게 참여할 수 있는지

### 주요 행동

- 피드백 남기기
- 테스터 신청
- 저장
- 맥락 기반 연락

### Builder가 공개 여부를 제어해야 하는 정보

- 협업 요청 내용
- 연락 가능 범위
- 테스터 모집 조건

### 비개발자 친화적 표현 방식

“기여”보다 “지금 필요한 도움”으로 표현한다.

### 초기 구현 요소

- 필요한 도움 표시
- 피드백 요청 연결

### 후순위 요소

- 테스터 신청 폼
- 협업자 요청 상세 조건

## 8. 문제 정의

### 표시 정보

- 현재 문제 정의
- 문제 정의가 최근 바뀌었는지 여부
- 문제 정의 수정 변화 카드 링크

### 사용자가 이해해야 하는 것

- 이 프로젝트가 어떤 문제에서 출발했는지

### 주요 행동

- 문제 정의에 피드백 남기기
- 관련 변화 카드 보기

### Builder가 공개 여부를 제어해야 하는 정보

- 내부 고객명
- 민감한 시장 정보
- 비공개 인터뷰 내용

### 비개발자 친화적 표현 방식

긴 분석 문서보다 “누가 어떤 상황에서 무엇 때문에 막히는가”로 보여준다.

### 초기 구현 요소

- 현재 문제 정의 표시

### 후순위 요소

- 문제 정의 변경 히스토리 공개

## 9. 현재 가설

### 표시 정보

- 현재 검증 중인 가설
- 가설 상태
- 관련 변화 카드

### 사용자가 이해해야 하는 것

- Builder가 현재 무엇을 믿고 실험 중인지

### 주요 행동

- 가설에 대한 피드백
- 테스트 참여

### 초기 구현 요소

- 핵심 가설 1~3개 표시

### 후순위 요소

- 전체 가설 목록
- 가설별 검증 히스토리

## 10. 최근 전환

### 표시 정보

- 가장 최근의 핵심 전환 변화 카드
- 전환 전 판단
- 전환 후 판단
- 전환 이유

### 사용자가 이해해야 하는 것

- 프로젝트가 단순히 업데이트된 것이 아니라 어떤 판단으로 바뀌었는지

### 주요 행동

- 전환 카드 보기
- Decision Diff 보기

### 초기 구현 요소

- 최근 핵심 전환 카드 요약

### 후순위 요소

- 여러 전환점 타임라인

## 11. 프로젝트 진화도

### 표시 정보

- 문제 발견
- 가설
- 실험
- 피드백
- 판단
- 방향 전환
- 현재 상태

### 사용자가 이해해야 하는 것

- 프로젝트가 어떤 길을 거쳐 현재에 도달했는지

### 주요 행동

- 핵심 전환 카드 열기
- Timeline 전체 보기

### 초기 구현 요소

- 핵심 변화 카드 중심의 간단한 흐름

### 후순위 요소

- 시각적 노드 구조
- 가설별 가지 구조

## 12. 공개 변화 카드

### 표시 정보

- 공개됨 상태의 변화 카드
- 핵심 전환 우선
- 일반 카드는 접힘 처리 가능

### 사용자가 이해해야 하는 것

- Builder가 실제로 어떤 판단을 했는지

### 주요 행동

- 카드 펼치기
- 피드백 남기기
- 관련 가설 보기

### Builder가 공개 여부를 제어해야 하는 정보

- 내부 전용 카드
- 민감 정보 포함 카드
- 아직 공개하면 안 되는 실험 결과

### 초기 구현 요소

- 공개 카드 목록
- 핵심 전환 강조

### 후순위 요소

- 공개 카드 필터
- 공개 카드 검색

## 13. 피드백 요청

### 표시 정보

- Builder가 현재 받고 싶은 피드백 질문
- 피드백 대상
- 원하는 피드백 유형

### 사용자가 이해해야 하는 것

- 무엇에 대해 답하면 Builder에게 도움이 되는지

### 주요 행동

- 피드백 작성
- 테스터 참여 의사 표시

### 초기 구현 요소

- 피드백 요청 카드
- 로그인 기반 피드백 작성 이동

### 후순위 요소

- 비로그인 피드백
- 피드백 투표

## 14. 관련 링크

### 표시 정보

- 데모 링크
- GitHub 링크
- Notion 링크
- 설문 링크
- 테스트 참여 링크

### 사용자가 이해해야 하는 것

- 자세한 결과물이나 외부 자료는 어디서 볼 수 있는지

### 주의점

관련 링크는 공개 페이지의 중심이 아니다. 중심은 판단 흐름이다.

## 15. Builder 정보

### 표시 정보

- Builder 이름 또는 표시명
- 역할
- 간단한 소개
- 공개 프로젝트 목록

### 사용자가 이해해야 하는 것

- 이 프로젝트를 성장시키는 사람이 누구인지

### 주의점

이력서보다 프로젝트 기록이 먼저다.

### 표시 정보

- 프로젝트 저장
- 업데이트 팔로우

### 알림 후보

- 새 공개 변화 카드
- 새 피드백 요청
- 테스터 모집
- 방향 전환

- 저장 또는 팔로우 버튼

### 후순위 요소

- 알림 설정
- 컬렉션 저장

- 현재 필요한 것

- 현재 가설
- 최근 전환
- 간단한 프로젝트 진화도

- Builder 정보
- 팔로우/저장

## 18. 후순위 요소

- 고급 프로젝트 진화도
- 공개 카드 필터
- Decision Diff 공개 포함

## 19. 추가 검토 필요

- 공개 페이지 상단에서 “현재 필요한 것”을 얼마나 강조할지
- 공개 카드가 적을 때 페이지 신뢰도를 어떻게 보완할지
- Builder 정보와 프로젝트 기록의 우선순위 배치
