# AI 변화 카드 초안 검토 및 승인 화면 텍스트 와이어

## 1. 화면 목적

이 화면은 AI가 거친 메모를 변화 카드 초안으로 구조화한 뒤, Builder가 내용을 검토하고 수정하며 최종 승인하는 화면이다.

BuildMap에서 변화 카드는 단순 로그가 아니라 의사결정 단위다. 따라서 이 화면은 BuildMap의 핵심 화면이다.

## 2. 기본 원칙

- AI는 기록을 꾸미지 않는다.
- AI는 없는 성과를 만들지 않는다.
- AI는 Builder의 입력을 문제, 근거, 판단, 변경, 다음 확인 사항으로 구조화한다.
- Builder가 최종 승인자다.
- 승인 전에는 Decision Timeline에 공식 반영하지 않는다.
- AI 초안은 자동 공개되지 않는다.

## 3. 진입 조건

- Builder가 거친 메모를 입력했다.
- AI 구조화가 완료되었다.
- 또는 Builder가 임시 저장된 메모에서 변화 카드 초안 생성을 실행했다.

## 4. AI 초안 결과 영역

### 표시 정보

- AI가 제안한 변화 카드 유형
- 제목
- 구조화 요약
- 근거
- 판단
- 변경 내용
- 다음 확인 사항
- 연결 후보 문제 또는 가설
- 공개 여부 기본값
- 중요도 제안

### 안내 문구

> 아래 내용은 AI가 구조화한 초안입니다. Builder가 수정하고 승인해야 공식 Decision Timeline에 반영됩니다.

## 5. Builder 수정 영역

Builder는 모든 항목을 수정할 수 있어야 한다.

### 수정 가능한 항목

- 변화 카드 유형
- 제목
- 원문 메모
- 구조화 요약
- 근거
- 판단
- 변경 내용
- 다음 확인 사항
- 연결된 문제 또는 가설
- 공개 여부
- 상태
- 중요도

### UX 원칙

AI 초안을 그대로 확정하도록 압박하지 않는다. Builder가 원래 의도와 다르면 수정하는 흐름이 자연스러워야 한다.

## 6. 변화 카드 유형 선택

### 유형 예시

- 문제 발견
- 문제 정의 수정
- 가설 생성
- 가설 반박
- 실험
- 사용자 피드백
- 기능 추가
- 기능 제거
- 판단 유지
- 판단 수정
- 방향 전환
- 릴리즈
- 인수인계 메모

### 도움말 문구

> 변화 카드 유형은 이 기록이 프로젝트에서 어떤 판단 단위인지 나타냅니다. 정확히 모르겠다면 나중에 수정할 수 있습니다.

## 7. 변화 카드 최소 정보

아래 항목은 제품 설계용 최소 필드로 가정한다. DB 스키마나 컬럼명으로 확정하지 않는다.

### 유형

이 카드가 어떤 의사결정인지 표시한다.

### 제목

Timeline에서 한눈에 볼 수 있는 짧은 제목이다.

### 원문 메모

AI 구조화 전 Builder가 입력한 실제 메모다.

### 구조화 요약

AI가 정리한 핵심 요약이다.

### 근거

판단의 근거가 되는 사용자 반응, 테스트 결과, 회의 내용, 관찰 내용이다.

### 판단

Builder가 어떤 결론을 내렸는지 표시한다.

### 변경 내용

실제로 바꾼 것, 제거한 것, 유지한 것, 보류한 것을 표시한다.

### 다음 확인 사항

다음 실험, 테스트, 관찰, 피드백 요청을 표시한다.

### 연결된 문제 또는 가설

가능하다면 현재 문제 정의나 가설과 연결한다.

### 공개 여부

내부 전용, 공개 가능, 공개됨 등의 상태를 선택한다.

### 상태

초안, 수정 중, 승인됨, 보류됨 중 하나로 표시한다.

### 중요도

일반 또는 핵심 전환으로 표시한다.

## 8. 상태 선택

| 상태 | 의미 |
|---|---|
| 초안 | 아직 Builder가 승인하지 않은 상태 |
| 수정 중 | Builder가 내용을 검토하고 수정 중인 상태 |
| 승인됨 | 공식 Decision Timeline에 반영된 상태 |
| 보류됨 | 지금은 Timeline에 반영하지 않기로 한 상태 |

## 9. 중요도 선택

| 중요도 | 의미 |
|---|---|
| 일반 | Timeline에는 표시되지만 핵심 전환으로 강조하지 않음 |
| 핵심 전환 | 프로젝트 진화도와 공개 페이지에서 강조할 수 있는 중요한 판단 |

## 10. 연결된 문제 또는 가설

### 원칙

- 변화 카드는 단독 생성 가능하다.
- 승인 시에는 문제 또는 가설과 연결하도록 유도한다.
- 연결을 강제하지는 않는다.

### 안내 문구

> 이 변화 카드가 특정 문제 정의나 가설과 연결된다면 선택해주세요. 연결하면 나중에 프로젝트가 왜 바뀌었는지 더 쉽게 볼 수 있습니다.

## 11. 공개 여부

### 선택 후보

- 내부 전용
- 공개 가능
- 공개됨
- 민감 정보 포함

### 기본값

AI 초안의 기본값은 내부 전용이 적합하다.

### 공개 전 확인 문구

> 공개하면 이 변화 카드가 공개 프로젝트 페이지에 노출될 수 있습니다. 민감한 내부 정보, 개인 정보, 확정되지 않은 외부 약속이 포함되어 있지 않은지 확인해주세요.

## 12. 주요 버튼

### 승인하고 Timeline에 반영

- 상태를 승인됨으로 변경한다.
- 공식 Decision Timeline에 반영한다.
- 공개 상태가 공개됨이면 공개 페이지에 노출될 수 있다.

### 수정 저장

- 현재 수정 내용을 저장한다.
- 아직 Timeline에 공식 반영하지 않을 수 있다.

### 보류

- 현재 초안을 보류 상태로 저장한다.
- 대시보드에서 나중에 다시 볼 수 있다.

### 삭제

- 초안을 삭제한다.
- 원문 메모 보존 여부는 추가 검토 필요.

## 13. 승인 전 경고

### 표시 조건

- 연결된 문제 또는 가설이 없음
- 근거가 비어 있음
- 판단이 비어 있음
- 공개됨 상태인데 민감 정보 포함 가능성이 있음

### 경고 문구 예시

> 이 변화 카드는 아직 근거가 비어 있습니다. 판단의 이유가 남아야 나중에 프로젝트 맥락을 이해할 수 있습니다.

> 공개 상태로 승인하려고 합니다. 외부에 보여도 되는 내용인지 확인해주세요.

## 14. AI가 없는 내용을 만들지 않았는지 확인하는 체크리스트

승인 전 Builder에게 다음을 확인하게 한다.

- AI가 실제로 말하지 않은 성과를 추가하지 않았는가?
- 사용자 수, 테스트 수, 피드백 수를 과장하지 않았는가?
- 판단의 근거가 원문 메모와 일치하는가?
- 변경 내용이 실제 작업 또는 결정과 맞는가?
- 공개하면 안 되는 내부 정보가 포함되어 있지 않은가?
- 다음 확인 사항이 실행 가능한가?
