# 거친 메모 입력 화면 텍스트 와이어

## 1. 화면 목적

이 화면은 Builder가 회의 후, 사용자 테스트 후, 작업 중, 아이디어 변경 직후에 떠오른 판단 재료를 빠르게 남기는 화면이다.

Builder는 처음부터 완성된 회고문이나 정리글을 쓰지 않는다. BuildMap은 거친 메모를 받아 AI가 변화 카드 초안으로 구조화하고, Builder가 이를 수정·승인하는 흐름을 제공한다.

## 2. 주요 사용자

- Builder
- 프로젝트를 운영 중인 팀원
- 사용자 테스트를 정리하는 사람
- 회의 내용을 판단 단위로 바꾸려는 사람

## 3. 진입 조건

- 프로젝트가 생성되어 있다.
- Builder 대시보드, 프로젝트 대시보드, Decision Timeline, 변화 카드 목록에서 진입할 수 있다.

## 4. 입력 방식

1차 입력 방식은 **텍스트 메모 하나**로 제한한다.

### 입력 필드

- 거친 메모

### 입력 예시

> 테스터 3명이 첫 화면이 너무 복잡하다고 했다. 10년 후 목표를 먼저 쓰라고 하니까 부담스럽다고 했다. 그래서 첫 화면에서는 이번 주 목표부터 고르게 바꾸는 게 맞을 것 같다.

> AI가 3개월 계획을 자동으로 만들어주는 기능을 뺐다. 계획이 너무 길어서 아무도 끝까지 안 읽었다.

> 한 사용자가 캘린더 연동이 필요하다고 했지만 지금은 핵심이 아니라고 판단했다. 아직 문제 정의 검증이 먼저다.

## 5. 입력 도움말

### 기본 안내

> 완성된 문장으로 쓰지 않아도 됩니다. 회의 내용, 사용자 반응, 기능을 바꾼 이유, 버린 이유를 그대로 적어주세요.

### 적어도 좋은 내용

- 사용자 피드백
- 테스트 결과
- 회의에서 나온 판단
- 기능 추가 이유
- 기능 제거 이유
- 방향 전환 계기
- 릴리즈 목적
- 인수인계 메모

### 피해야 할 부담

- 완성된 블로그 글처럼 쓰기
- 성과를 과장하기
- 모든 근거를 완벽하게 정리하기

## 6. 주요 버튼

### AI로 변화 카드 초안 만들기

- 입력된 텍스트를 AI 구조화 단계로 보낸다.
- 결과는 바로 공개되지 않는다.
- 결과는 Builder 승인 전까지 공식 Decision Timeline에 반영되지 않는다.

### 임시 저장

- 메모를 원문 상태로 저장한다.
- 나중에 다시 구조화할 수 있다.

### 취소

- 입력을 중단한다.
- 입력 중인 내용이 있으면 확인 메시지를 표시한다.

## 7. AI 구조화 전 원문 보관 안내

안내 문구:

> AI가 구조화하더라도 원문 메모는 보관됩니다. 나중에 AI 초안이 원래 의도를 왜곡하지 않았는지 확인할 수 있습니다.

## 8. 공개되지 않는다는 안내

안내 문구:

> 이 메모는 바로 공개되지 않습니다. AI 초안도 Builder가 승인하고 공개 여부를 선택하기 전에는 외부에 노출되지 않습니다.

## 9. 빈 상태

### 상황

입력칸이 비어 있다.

### 메시지

> 지금 떠오른 판단을 짧게 적어보세요. “왜 바꿨는지”만 남겨도 충분합니다.

### 가능한 행동

- 예시 보기
- 최근 피드백에서 메모 시작
- 취소

## 10. 입력 중 상태

### 표시 정보

- 글자 수 또는 대략적인 길이 안내
- 저장되지 않은 상태 표시
- AI 구조화 가능 여부

### UX 원칙

짧은 메모도 허용한다. 지나치게 긴 입력을 요구하지 않는다.

## 11. AI 처리 중 상태

### 메시지

> 메모를 문제, 근거, 판단, 변경 내용, 다음 확인 사항으로 구조화하고 있습니다.

### 표시 원칙

AI가 “생성”하고 있다는 느낌보다 “구조화”하고 있다는 표현을 사용한다.

## 12. AI 처리 실패 상태

### 메시지

> 메모를 변화 카드 초안으로 구조화하지 못했습니다. 원문 메모는 유지되어 있습니다.

### 가능한 행동

- 다시 시도
- 직접 변화 카드 작성
- 임시 저장
- 취소

## 13. 초기 구현 요소

- 텍스트 메모 입력
- AI로 변화 카드 초안 만들기
- 임시 저장
- 취소
- 원문 보관 안내
- 자동 공개되지 않음 안내
- AI 처리 중/실패 상태

## 14. 후순위 요소

- 음성 입력
- 이미지 입력
- 파일 업로드
- 회의록 업로드
- GitHub/Notion 연동
- Slack/카카오톡 대화 가져오기
- 여러 메모를 묶어 변화 카드 생성

## 15. 추가 검토 필요

- 원문 메모를 변화 카드와 어떻게 함께 보여줄지
- AI 구조화 실패 시 재시도 횟수 제한이 필요한지
- 메모만 저장된 항목을 대시보드에서 어떻게 노출할지
