# 피드백 요청 화면 텍스트 와이어

## 1. 화면 목적

피드백 요청 화면은 Builder가 프로젝트 전체에 대한 막연한 평가를 요청하는 화면이 아니다.

이 화면은 특정 문제, 가설, 변화 카드, 공개 페이지에 대해 어떤 피드백이 필요한지 명확히 묻는 화면이다.

## 2. 주요 사용자

- Builder
- 프로젝트 팀원

## 3. 진입 조건

- 프로젝트가 생성되어 있다.
- 문제 정의, 가설, 변화 카드, 공개 프로젝트 페이지 중 하나에 연결하여 진입할 수 있다.

## 4. 피드백 요청 대상

### 선택 후보

- 프로젝트 전체
- 문제 정의
- 가설
- 변화 카드
- 공개 페이지

### 기본 원칙

초기에는 프로젝트 전체보다 특정 판단 대상에 피드백을 연결하는 흐름을 우선한다.

### 안내 문구

> 어떤 판단에 대해 피드백이 필요한지 선택하면 더 좋은 답변을 받을 수 있습니다.

## 5. 피드백 질문 입력

### 입력 필드

- 피드백 질문

### 도움말 문구

> “어떤가요?”보다 “첫 화면에서 서비스 목적이 이해되나요?”처럼 구체적으로 묻는 것이 좋습니다.

### 예시 문구

- 이 문제 정의가 실제로 공감되나요?
- 이 가설이 약해 보이는 부분이 있나요?
- 첫 화면에서 무엇을 해야 하는지 이해되나요?
- 이 기능을 제거한 판단에 동의하나요?
- 이 프로젝트에 테스터로 참여하고 싶으신가요?

## 6. 원하는 피드백 유형 선택

### 유형 후보

- 공감 여부
- 이해도
- 기능 필요성
- 사용 경험
- 개선 제안
- 테스터 참여

### 선택 방식

- 복수 선택 가능
- 기본값은 선택하지 않아도 되지만, 선택을 권장한다.

### 설명

피드백 유형은 응답자가 어떤 관점으로 답해야 하는지 알려준다.

## 7. 공개 여부

### 선택 후보

- 내부 요청
- 공개 프로젝트 페이지에 표시
- 링크를 아는 사람만 응답 가능

### 기본값

프로젝트 공개 상태에 따라 다르지만, 안전하게 내부 또는 링크 공개를 기본값으로 두는 것이 적합하다.

### 안내 문구

> 공개 피드백 요청은 외부 사용자에게 노출될 수 있습니다. 아직 공개하면 안 되는 정보가 포함되어 있지 않은지 확인해주세요.

## 8. 요청 상태

### 상태 후보

- 임시 저장
- 공개 중
- 종료됨
- 보류됨

### 상태별 의미

- 임시 저장: 아직 외부에 노출되지 않음
- 공개 중: 응답을 받을 수 있음
- 종료됨: 더 이상 응답을 받지 않음
- 보류됨: 지금은 사용하지 않음

## 9. 주요 버튼

### 피드백 요청 공개

- 공개 상태에 따라 프로젝트 페이지 또는 링크에 노출한다.

### 임시 저장

- 요청을 저장하지만 외부에 노출하지 않는다.

### 취소

- 작성 중인 요청을 중단한다.

## 10. 초기 구현 요소

- 피드백 대상 선택
- 피드백 질문 입력
- 원하는 피드백 유형 선택
- 공개 여부 선택
- 요청 상태
- 공개/임시 저장/취소

## 11. 후순위 요소

- 응답 기한
- 대상 사용자 지정
- 피드백 템플릿
- 피드백 요청 반복 발행
- Scout 그룹에만 공개
- 피드백 품질 가이드

## 12. 추가 검토 필요

- 프로젝트 전체 피드백 요청을 얼마나 허용할지
- 피드백 요청 종료 후 결과를 자동으로 변화 카드 초안으로 제안할지
- 공개 피드백 요청이 많아질 때 프로젝트 페이지에서 노출 순서
