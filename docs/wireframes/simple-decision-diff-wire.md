# 간단한 Decision Diff 화면 텍스트 와이어

## 1. 화면 목적

Decision Diff는 코드 diff가 아니다.

Decision Diff는 프로젝트의 이전 판단과 현재 판단이 어떻게 달라졌는지 비교하는 화면이다. BuildMap에서 중요한 것은 파일 변경이 아니라 문제 정의, 가설, 타깃 사용자, 우선순위, 성공 기준의 변화다.

## 2. 주요 사용자

- Builder
- Scout
- 새 팀원
- 협업자
- 멘토
- 심사위원

## 3. 비교 대상

### 초기 판단 vs 현재 판단

프로젝트를 처음 시작했을 때의 판단과 현재 판단을 비교한다.

### 전환 전 vs 전환 후

특정 방향 전환 변화 카드 기준으로 비교한다.

### 특정 시점 A vs 특정 시점 B

Timeline의 두 시점을 선택해 비교한다.

초기 구현에서는 “초기 판단 vs 현재 판단” 또는 “전환 전 vs 전환 후”를 우선한다.

## 4. 비교 항목

### 문제 정의

- 이전 문제 정의
- 현재 문제 정의
- 변경 이유
- 연결 변화 카드

### 핵심 가설

- 이전 가설
- 현재 가설
- 상태 변화
- 반박 또는 확인 근거

### 타깃 사용자

- 이전 타깃
- 현재 타깃
- 전환 이유

### 핵심 기능

- 유지한 기능
- 제거한 기능
- 추가한 기능
- 보류한 기능

### 우선순위

- 이전 우선순위
- 현재 우선순위
- 바뀐 이유

### 성공 기준

- 이전 성공 기준
- 현재 성공 기준
- 검증 방법 변화

## 5. 표시 방식

### 좌우 비교

```text
이전 판단                         현재 판단
사용자는 10년 목표가 필요하다  →  사용자는 이번 주 실행 증거가 필요하다
```

### 변경 이유

각 비교 항목 아래에 변경 이유를 표시한다.

예시:

> 테스트 참여자들이 장기 목표 입력을 부담스럽게 느꼈기 때문에 첫 진입 흐름을 단순화했다.

### 연결된 변화 카드

변경 이유는 관련 변화 카드로 연결한다.

표시 예시:

- [가설 반박] 장기 목표 선입력 흐름 제거
- [사용자 피드백] 첫 화면 복잡도 피드백 반영

## 6. 주요 버튼

### 비교 기준 선택

- 초기 판단 vs 현재 판단
- 전환 전 vs 전환 후
- 시점 선택

### 연결된 변화 카드 보기

- 해당 판단 변화의 근거가 된 카드로 이동

### 공개 페이지에 포함

- 공개 가능한 Decision Diff를 공개 프로젝트 페이지에 포함
- 공개 전 민감 정보 확인 필요

## 7. 초기 구현 요소

- 초기 판단 vs 현재 판단 비교
- 전환 전 vs 전환 후 비교
- 문제 정의 비교
- 핵심 가설 비교
- 핵심 기능 비교
- 변경 이유 표시
- 연결 변화 카드 표시

## 8. 후순위 요소

- 특정 시점 A/B 자유 선택
- 공개 페이지용 Diff 카드
- 여러 가설 동시 비교
- 시각적 diff 뷰
- 팀원별 판단 기여 이력

## 9. 추가 검토 필요

- 초기 판단을 어떤 시점으로 정의할지
- 비교 항목을 Builder가 직접 선택할 수 있게 할지
- 공개 가능한 Diff와 내부 Diff를 어떻게 분리할지
