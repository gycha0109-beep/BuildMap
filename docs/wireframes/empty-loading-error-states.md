# 공통 빈 상태·로딩·오류 상태 문서

## 1. 문서 목적

이 문서는 BuildMap의 주요 화면에서 공통으로 발생하는 빈 상태, 로딩 상태, 오류 상태, 권한 상태, 공개 전환 경고를 정리한다.

BuildMap의 상태 메시지는 단순 시스템 문구가 아니라 Builder가 판단 흐름을 계속 이어갈 수 있도록 도와야 한다.

## 2. 프로젝트가 없는 상태

### 상황

Builder가 가입했지만 아직 프로젝트를 만들지 않았다.

### 사용자에게 보여줄 메시지

> 아직 프로젝트가 없습니다. 완성작이 없어도 괜찮습니다. 지금 만들고 있는 아이디어나 문제의식부터 기록해보세요.

### 가능한 행동

- 첫 프로젝트 만들기
- 예시 프로젝트 보기

### 피해야 할 표현

- 포트폴리오를 등록하세요
- 완성된 제품을 업로드하세요

### 초기 구현 요소

- 첫 프로젝트 만들기 버튼

### 후순위 요소

- 예시 프로젝트 추천

## 3. 문제 정의가 없는 상태

### 상황

프로젝트는 있지만 문제 정의가 비어 있다.

### 메시지

> 아직 문제 정의가 없습니다. BuildMap에서는 프로젝트가 어떤 문제에서 시작했는지가 가장 중요한 첫 기록입니다.

### 가능한 행동

- 문제 정의 입력
- 예시 보기

### 피해야 할 표현

- 필수 항목이 누락되었습니다

## 4. 가설이 없는 상태

### 상황

프로젝트에 등록된 가설이 없다.

### 메시지

> 아직 가설이 없습니다. 이 프로젝트가 성립하려면 무엇이 사실이어야 하는지 적어보세요.

### 가능한 행동

- 첫 가설 추가
- 지금은 건너뛰기

### 피해야 할 표현

- 기능 목록을 입력하세요

## 5. 변화 카드가 없는 상태

### 상황

아직 승인된 변화 카드가 없다.

### 메시지

> 아직 변화 카드가 없습니다. 회의 내용, 사용자 반응, 기능을 바꾼 이유를 거친 메모로 남기면 변화 카드 초안으로 구조화할 수 있습니다.

### 가능한 행동

- 거친 메모 입력
- 직접 변화 카드 작성

## 6. AI 초안 생성 중

### 상황

거친 메모를 AI가 변화 카드 초안으로 구조화 중이다.

### 메시지

> 메모를 문제, 근거, 판단, 변경 내용, 다음 확인 사항으로 구조화하고 있습니다.

### 가능한 행동

- 기다리기
- 취소는 추가 검토 필요

### 피해야 할 표현

- 멋진 포트폴리오 문장을 생성 중입니다

## 7. AI 초안 생성 실패

### 상황

AI 구조화가 실패했다.

### 메시지

> 메모를 변화 카드 초안으로 구조화하지 못했습니다. 원문 메모는 유지되어 있습니다.

### 가능한 행동

- 다시 시도
- 직접 작성
- 임시 저장

## 8. 피드백이 없는 상태

### 상황

피드백 요청은 있으나 아직 응답이 없다.

### 메시지

> 아직 피드백이 없습니다. 질문이 구체적일수록 좋은 답변을 받을 가능성이 높습니다.

### 가능한 행동

- 피드백 질문 수정
- 공개 페이지에서 요청 보기
- 요청 종료

## 9. 공개 가능한 카드가 없는 상태

### 상황

프로젝트 공개 페이지를 만들려 하지만 공개 가능한 변화 카드가 없다.

### 메시지

> 아직 공개 가능한 변화 카드가 없습니다. 내부 기록 중 외부에 보여도 되는 판단 흐름을 선택해보세요.

### 가능한 행동

- 변화 카드 공개 검토
- 내부 전용 유지
- 공개 페이지 임시 저장

## 10. Timeline이 비어 있는 상태

### 상황

Decision Timeline에 표시할 승인 카드가 없다.

### 메시지

> 아직 Decision Timeline에 반영된 판단 기록이 없습니다. 첫 변화 카드를 승인하면 프로젝트의 변화 흐름이 시작됩니다.

### 가능한 행동

- 거친 메모 입력
- AI 초안 보기
- 변화 카드 직접 작성

## 11. 권한이 없어 볼 수 없는 상태

### 상황

사용자가 내부 전용 프로젝트 또는 내부 전용 변화 카드에 접근했다.

### 메시지

> 이 기록은 Builder가 공개하지 않은 내부 기록입니다.

### 가능한 행동

- 공개 프로젝트 페이지로 이동
- 접근 요청은 후순위

### 피해야 할 표현

- 접근 실패
- 오류 발생

## 12. 비공개 프로젝트 접근 상태

### 상황

비공개 프로젝트 링크에 외부 사용자가 접근했다.

### 메시지

> 이 프로젝트는 아직 공개되지 않았습니다.

### 가능한 행동

- 탐색 화면으로 돌아가기
- 로그인 계정 확인

## 13. 네트워크 오류

### 상황

데이터를 불러오지 못했다.

### 메시지

> 프로젝트 판단 흐름을 불러오지 못했습니다. 잠시 후 다시 시도해주세요.

### 가능한 행동

- 다시 시도
- 대시보드로 이동

## 14. 저장 실패

### 상황

메모, 변화 카드, 피드백, 공개 설정 저장이 실패했다.

### 메시지

> 저장하지 못했습니다. 입력한 내용은 화면에 유지되어 있습니다. 다시 시도해주세요.

### 가능한 행동

- 다시 저장
- 임시 복사
- 취소

### 원칙

작성한 판단 기록이 사라졌다는 느낌을 주면 안 된다.

## 15. 공개 전환 확인

### 상황

Builder가 프로젝트 또는 변화 카드를 공개하려고 한다.

### 메시지

> 공개하면 선택한 정보가 외부 사용자에게 보일 수 있습니다. 공개할 변화 카드와 민감한 정보를 확인해주세요.

### 가능한 행동

- 공개하기
- 미리보기
- 계속 비공개

## 16. 민감 정보 공개 경고

### 상황

변화 카드 또는 공개 페이지에 민감할 수 있는 정보가 포함되어 있다.

### 메시지

> 이 기록에는 내부 정보나 개인 정보가 포함되어 있을 수 있습니다. 공개 전에 내용을 다시 확인해주세요.

### 가능한 행동

- 수정하기
- 내부 전용 유지
- 그래도 공개

### 후순위 요소

- 자동 민감 정보 감지
- 공개 전 체크리스트 자동화

## 17. 초기 구현 요소

- 프로젝트 없음
- 문제 정의 없음
- 가설 없음
- 변화 카드 없음
- AI 생성 중
- AI 생성 실패
- 피드백 없음
- 공개 가능 카드 없음
- Timeline 비어 있음
- 권한 없음
- 비공개 프로젝트
- 네트워크 오류
- 저장 실패
- 공개 전환 확인
- 민감 정보 공개 경고

## 18. 후순위 요소

- 예시 데이터 삽입
- 상태별 튜토리얼
- 자동 복구
- 임시 저장 복원
- 민감 정보 자동 탐지
- 접근 요청 플로우

## 19. 추가 검토 필요

- 비공개 프로젝트 접근 시 Builder에게 알릴지
- 저장 실패 시 로컬 임시 보존을 어떻게 처리할지
- 민감 정보 경고를 수동 체크리스트로 둘지 자동 감지로 확장할지
