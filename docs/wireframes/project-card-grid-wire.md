# 프로젝트 카드 그리드 탐색 화면 텍스트 와이어

## 1. 화면 목적

프로젝트 카드 그리드는 BuildMap의 초기 탐색 화면이다.

히트맵은 시각적으로 좋지만 지표 산식이 아직 불안정하다. 따라서 1차 탐색 화면은 카드 그리드로 시작하고, 히트맵은 후순위 실험 화면으로 둔다.

## 2. 왜 히트맵보다 카드 그리드가 먼저인가

- 카드 그리드는 지표 산식이 없어도 작동한다.
- 프로젝트의 문제 정의, 현재 필요한 것, 최근 전환을 명확히 보여줄 수 있다.
- 비개발자도 읽기 쉽다.
- Scout가 피드백이나 테스트 참여로 이어지기 쉽다.
- 히트맵은 활동성 기준이 안정된 뒤 실험하는 것이 적합하다.

## 3. 카드에 표시할 정보

### 프로젝트명

프로젝트 식별 정보다.

### 한 줄 정의

프로젝트가 무엇을 하려는지 짧게 보여준다.

### 현재 상태

완성도가 아니라 판단 단계로 표시한다.

예시:

- 문제 정의 중
- 가설 검증 중
- 테스트 중
- 방향 전환 중

### 현재 필요한 것

Scout가 즉시 행동할 수 있는 정보다.

예시:

- 문제 정의 피드백 필요
- 첫 화면 이해도 테스트 필요
- 테스터 모집 중
- 협업자 필요

### 최근 전환

프로젝트가 최근 어떻게 바뀌었는지 보여준다.

예시:

> 장기 목표 입력을 제거하고 이번 주 실행 단위를 먼저 보여주기로 전환

### 최근 활동

- 최근 변화 카드 작성일
- 최근 공개 업데이트
- 최근 피드백 요청

### 배지

- 테스터 모집
- 피드백 요청
- 첫 공개
- 최근 전환
- 베타
- 협업자 필요

### Builder

- Builder 표시명
- 역할 또는 간단 소개

## 4. 배지 예시

| 배지 | 의미 |
|---|---|
| 테스터 모집 | 현재 테스트 참여자를 찾고 있음 |
| 피드백 요청 | 특정 판단에 대한 피드백을 받고 있음 |
| 첫 공개 | 최근 처음으로 공개된 프로젝트 |
| 최근 전환 | 최근 핵심 전환 변화 카드가 있음 |
| 베타 | 외부 사용자가 일부 사용 가능함 |
| 협업자 필요 | 특정 역할의 협업자를 찾고 있음 |

## 5. 정렬 기준 후보

### 최근 업데이트

가장 단순한 기본 정렬이다.

### 피드백 요청 중

Scout가 바로 참여할 수 있는 프로젝트를 먼저 보여준다.

### 테스터 모집 중

테스터 참여 가능성이 높은 프로젝트를 먼저 보여준다.

### 최근 전환 있음

판단 흐름이 활발하게 움직인 프로젝트를 보여준다.

## 6. 주요 행동

- 프로젝트 보기
- 피드백 남기기
- 테스터 신청
- 저장
- Builder에게 맥락 기반 연락

## 7. 히트맵으로 확장 가능한 지점

카드 그리드에서 안정적으로 쌓이는 내부 활동 지표는 나중에 히트맵으로 확장할 수 있다.

확장 후보 지표:

- 최근 변화 카드 수
- 공개 업데이트 수
- 피드백 요청 수
- 피드백 응답 수
- 테스터 신청 수
- 저장 수
- 최근 핵심 전환 여부

단, 산식은 아직 확정하지 않는다.

## 8. 초기 구현 요소

- 공개 프로젝트 카드 그리드
- 기본 정렬
- 배지
- 프로젝트 보기
- 저장
- 피드백 요청 연결
- 테스터 모집 표시

## 9. 후순위 요소

- 히트맵 전환 보기
- 고급 활동성 지표
- 개인화 추천
- 분야별 컬렉션
- Scout 관심 분야 기반 추천

## 10. 추가 검토 필요

- 카드에서 최근 전환을 얼마나 길게 표시할지
- 배지가 많을 때 우선순위
- 정렬 기준의 기본값
- 히트맵 실험 시점
